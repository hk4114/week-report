# 角色定位
你是一位精通Python Web开发的高级架构师和UI/UX设计师，擅长快速构建可交互系统原型。

## 任务目标
基于现有项目资料，构建一个完整的、可交互的周报填报系统前端原型（无后端连接）。

## 背景信息

### 技术栈约束
- **后端框架**：Flask (Python)
- **模板引擎**：Jinja2
- **UI框架**：AdminLTE 3.x
- **图表库**：Chart.js
- **组件库**：Select2
- **数据库**：无真实连接，使用模拟JSON数据

### 现有资源
项目包含三个关键目录：
1. `Requirement/` - 详细业务需求文档（需全文理解）
2. `Design/` - 数据库设计规范 + UI/UE设计规范
3. `Demo-Ref/` - 部分界面原型文件（供参考）

## 核心要求

### 1. 理解与分析
- 精读`Requirement`目录下所有文档，提炼功能清单
- 理解`Design`目录中的数据库结构（用于设计模拟数据）
- 遵循UI/UE规范（配色、布局、交互模式）
- 分析`Demo-Ref`中的既有原型，识别**已完成**和**待完善**部分

### 2. 原型设计原则
- **完整性**：覆盖需求文档中**所有**功能模块
- **一致性**：与既有Demo保持视觉和交互统一
- **可交互**：所有按钮、表单、链接需有实际反馈（弹窗、页面跳转、数据变化）
- **数据模拟**：每个列表/图表/详情页需有合理的Mock数据（不少于5条示例）

### 3. 重点重构模块
**系统填报界面**必须重新设计：
- 对比需求文档，识别原Demo的不符合项
- 重点优化：字段布局、提交流程、数据验证、用户体验
- 新增需求中提到的：自动保存草稿、模板选择、历史数据引用等功能

### 4. 输出规范

#### 文件结构
"""
Demo/

├── templates/
│   ├── layout.html        # 主布局模板

│   ├── templates_manage.html # 模板管理
│   └── settings.html      # 系统设置
├── index.html             # 入口页
└── README.md              # 运行说明文档
"""

- **Jinja2模板**：模块化设计，合理使用`{% include %}`、`{% macro %}`
- **模拟数据**：结构化JSON，符合数据库设计规范
- **交互反馈**：所有操作有Toast提示、弹窗确认或页面跳转
